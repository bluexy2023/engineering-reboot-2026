# Git Information

## Repository
- a repository is a project directory that is managed by git which contains the files and the versions (or who reviesed them and when), as well as the changes that were made to certain files or directories under that project directory.  Timestamps are provided as well as information about the users who made the revisions.  

## Commit
- when someone uploads, or applies the changes made to a file or a directory within a repository

## Branch
- is a spin-off of an existing repository which is independent of the main branch.  This is usually being done to allow for making experimental changes to the files (prototypes/proof of concept).  By creating a branch, the main contents of a reposity remains unchanged, until a merge is performed.

## Merge
- the act of 'committing' changes from a branch onto a particular branch or to the main reposity.  This is usually being done once changes are proven to be acceptable.

## Remote
- remote as opposed to local, means your version of your repository resides somewhere else (cloud, github, other non-local repositories).  It is considered remote if it is not local to your laptop or desktop.

## Github
- Github is an online (cloud-based) platform that hosts git repositories.  While git is the version control system, Github is the service that stores and manages git repositories 'remotely'.  Other users with persmissions can access repositories on Github, which enables other users to collaborate on the revisions of the files being hosted on it.
