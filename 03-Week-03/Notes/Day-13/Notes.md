## Day 13 Notes

# Concepts Learned
- dictionaries are variable types that hold key and value pairs.  in C, it is similar to a structure data structure, which defines an entity with its corresponding attributes
- key are different from indexes in that the latter is numeric-based, while keys are arbitrary, so any string could be an attribute name or keys, or in the lingo of lists, the index is the key string, which describes an attribute of a particular object
- dictionaries should be used when one is describing an object and its unique attributes, and to be able to store characteristics of an object. lists on the other hand can be used to store multiple objects in an array, which could be accessed through its numeric index
- dictionaries model real-world data in that it allows for describing attributes of the data being represented. for example, if we are describing a car, one would ask many things about it; i.e, what's the make, is it a sedan or an SUV, what is its color, mileage, AWD, or 2WD?, etc
# Key Commands
- use of curly braces to define dictionaries - '{}'
- using strings as an index, ex: person["name"]
- print()
# Key Observations
- values in a dictionary is defined using a key-value pair, enclosed in curly brackets.  contents of dictionaries are accessed similar to a list, but using a "string" as index, where the "string" is the attribute name defined for that object, i.e, 'object["key"], resolves to a value
- lists on the other hand are defined using braces, '[]', where values are accessed using integer indexes
# Lessons Learned
- dictionaries is a convenient way to store data and its attributes.  the variable now hold much more data in that not only it stores the object, but allows us to define attributes for it.  Dictionaries and lists then can be used together to define a collection of objects with different attributes, for example, a list of students, with different names, course, age, and, grades
