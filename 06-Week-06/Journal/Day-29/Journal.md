# Reflection
 - git allows for managing the state of the repository, gives guidance on changed file as to whether to 
   stage them, ignore, or postpone them.  It allows for reverting or "restoring" local files into the version
   from previous commits, and it keeps a history of every version of the files or repositories committed to it.
 - commands are available to be able to restore, revert, or discard changes to a file or repository.  One must just
   keep on practicing them and apply their best engineering judgment whether to accept or reject changes to the repository
 - one way that organizations keep the sanity of its code repositories is through a buddy system where someone reviews the 
   changes to file(s) prior to committing them to the library.  If it is source code that they need to commit, a rudimentary   compile, test is performed as well as analyzing code for code standards.
