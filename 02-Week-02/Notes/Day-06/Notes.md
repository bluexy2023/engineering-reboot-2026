# Day 06 study notes

#Concepts Learned
Today I learned the concepts of decisions in python using integer values that fall a certain range or exactly a certain value.  Python is capable of allowing single decisions, double or multiple

The use of >= is to evaluate if a certain value is at least that value. Ex. if (age >= 18), it means, if age is at least 18

The use of <= is to evaluate if a certain value is at most that value.  Ex. if (score <= 100), it means if score is at most 100.

#Key Commands
- if: evaluates if a statement is true or false.  If TRUE, then the statement underneath that decision block gets executed
- else:  if in the first command, the evaluation of the expression yeilds a 'FALSE' result, the code under this decision block gets executed
- elif:  if statement evaluated by the 'if' statement is false, the code underneath this decision block is alternatively executed if the statement within it is true.  Multiple 'elif's can be stacked after another to evaluate alternative paths of the decision tree, until all statements hold false, then the code under the 'else:' decision block is executed.

#Key Observations
- in coding in python, indentation is crucial.  a decision block under the 'conditionals' (if, elsif, else) needs to be grouped under it using an indentation.  All code that aligns with it gets executed in turn sequentially

#Lessons Learned
- the use of IDEs like VSCode is helpful in that it allows for automatic indentation and it helps one to learn quicker.
